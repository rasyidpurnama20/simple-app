# Tutorial Aktor

Tutorial ini mengikuti empat peran utama aplikasi. Semua langkah menggunakan fitur yang tersedia saat ini: autentikasi, scoped assignment, registry rule/package, decision explanation dan replay, override/banding, integrity issue, dashboard capaian, katalog, tugas, kemajuan pribadi, dan Semantic Analytics API.

| Aktor | Fokus utama | Tutorial |
|---|---|---|
| Prodi | rule/package, maker-checker, keputusan dan approval | [Tutorial Prodi](prodi.md) |
| GPM | review rule, verifikasi issue, replay dan banding | [Tutorial GPM](gpm.md) |
| Pengampu | explanation/evidence, koreksi sumber, pengajuan override | [Tutorial Pengampu](pengampu.md) |
| Mahasiswa | keputusan pribadi, langkah perbaikan dan banding | [Tutorial Mahasiswa](mahasiswa.md) |

Fungsi DPA, koordinator, pembimbing, penguji, mentor, dan TPMF merupakan assignment terbatas, bukan akun utama. Assignment tersebut mengikuti scope objek, periode, aksi, dan expiry dari akun utama yang ditugaskan.
